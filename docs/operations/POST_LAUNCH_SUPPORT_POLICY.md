---

# 1. Purpose

This document defines the scope, limitations, and structure of post-launch support and maintenance services provided after the website is deployed to production.

---

# 2. Scope of Basic Support

The Basic Support Plan includes:

- Monitoring website availability
- Troubleshooting production issues
- Maintaining server configuration
- Periodic dependency updates (security and stability)
- SSL and domain checks
- Minor bug fixes
- MongoDB health checks
- Sitemap and robots verification

---

# 3. What Is Not Included

The following are NOT included in Basic Support:

- New feature development
- Major UI redesign
- Structural changes
- Content creation
- Third-party integration reconfiguration
- Infrastructure migration
- Performance scaling architecture
- Major version upgrades requiring refactor

These services require a separate agreement.

---

# 4. Support Model

Support is provided on a monthly retainer basis.

Example structure:

- Monthly Support Fee: $120
- Includes up to 4 hours per month
- Additional hours billed at $30/hour

Unused hours do not roll over to the next month.

---

# 5. Response Time

Critical Issues (site unavailable):

- Response within 24 hours

Major Issues (partial functionality failure):

- Response within 1–2 business days

Minor Issues:

- Addressed in next maintenance cycle

---

# 6. Client Responsibilities

The Client agrees to:

- Provide secure access credentials when needed
- Not share admin credentials publicly
- Notify the Developer promptly of issues
- Maintain hosting and third-party service payments

The Developer is not responsible for:

- Failures of hosting provider
- Failures of external services (email platforms, payment processors)
- Security breaches caused by credential misuse
- Content errors entered by admin users

---

# 7. Security & Maintenance

Maintenance includes:

- Reviewing dependency vulnerabilities
- Applying minor/patch updates
- Monitoring database health
- Ensuring public filtering rules remain enforced
- Verifying environment variables integrity

Major version upgrades are not included unless agreed separately.

---

# 8. Termination

Either party may terminate support with written notice.

Upon termination:

- Developer responsibility for uptime ends
- Access credentials must be revoked
- Client assumes operational responsibility

---

# 9. Disclaimer

While reasonable efforts are made to maintain system stability, no software system can guarantee zero downtime.

The Developer provides maintenance services in good faith and according to professional standards.

---

# 10. Acceptance

By continuing to use the deployed system under a support agreement, the Client acknowledges and agrees to this policy.
